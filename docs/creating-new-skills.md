# Creating New Skills

This guide helps you create high-quality, reusable skills for the collection.

## Step-by-Step

1. **Choose a focused scope**  
   One clear capability or domain (e.g. "research synthesis", not "everything about research").

2. **Use the template**  
   Copy `templates/new-skill-template.md` as starting point.

3. **Write the SKILL.md**
   - Strong frontmatter `name` and `description` (the description is critical — it determines when the skill activates).
   - Imperative instructions ("Do X", "Follow this workflow").
   - Concrete examples and templates where helpful.
   - Keep it concise.

4. **Test it**
   - Use the skill in real conversations.
   - Refine based on results.

5. **Submit PR**
   - Clear title and description of the skill's value.
   - Example use cases in the PR description.

## Quality Checklist
- [ ] Focused on one area
- [ ] Universal where possible (not overly tied to one model)
- [ ] Includes practical workflows or templates
- [ ] Examples are concrete and copy-paste ready
- [ ] SKILL.md < 500 lines (move long content to references/ if needed)
- [ ] Follows frontmatter + imperative style

## Naming
- Use kebab-case (e.g. `research-assistant`)
- Descriptive but short

Thank you for contributing to the skill ecosystem!
